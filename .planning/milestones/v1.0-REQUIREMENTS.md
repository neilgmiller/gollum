# Requirements: Gollum CLI Enhancements

**Defined:** 2026-04-02
**Core Value:** The wiki seamlessly follows the developer's local git workflow — serving the branch they're on and attributing edits to them.

## v1 Requirements

### Branch Tracking

- [x] **BRANCH-01**: User can set `wiki_options[:track_current_branch]` in config file or pass `--track-current-branch` CLI flag to serve pages from whatever branch is currently checked out
- [x] **BRANCH-02**: When `track_current_branch` is active, the served branch resolves from HEAD on each request (no caching)
- [x] **BRANCH-03**: If both `ref` and `track_current_branch` are set (via config file or CLI), gollum exits with a clear error message explaining they are mutually exclusive
- [x] **BRANCH-04**: If HEAD is detached (not pointing to a branch), gollum handles it gracefully — either errors with a clear message or falls back to the detached SHA

### Local Git User

- [x] **USER-01**: User can set `wiki_options[:local_git_user]` in config file or pass `--local-git-user` CLI flag to use local git config's user.name and user.email as commit author for web edits
- [x] **USER-02**: When `local_git_user` is active, git config is read fresh on each commit (not cached at startup)
- [x] **USER-03**: If git config user.name or user.email is empty/unset, gollum falls back gracefully with a warning rather than crashing

### Configuration

- [x] **CONF-01**: Both options work identically whether set via config file (`wiki_options[:track_current_branch]`, `wiki_options[:local_git_user]`) or CLI flags (`--track-current-branch`, `--local-git-user`)
- [x] **CONF-02**: Mutual exclusion validation runs after config file loading (not just after CLI parsing), since the config file can set `ref` or `track_current_branch`

## v2 Requirements

### Branch Tracking

- **BRANCH-05**: Handle bare repo HEAD resolution when `--track-current-branch` is combined with `--bare`
- **BRANCH-06**: Support `.git` file (worktree/submodule) layouts for HEAD resolution

### Local Git User

- **USER-04**: Session author (`session['gollum.author']` from rack middleware) takes priority over `--local-git-user`
- **USER-05**: Patch both commit paths (`commit_options()` and `upload_file` handler) for complete author injection

### UX

- **UX-01**: Log which branch is being served on each request when `--track-current-branch` is active
- **UX-02**: Startup validation warnings (e.g., "track-current-branch active, will follow HEAD")

## Out of Scope

| Feature | Reason |
|---------|--------|
| Background polling/file-watching for branch changes | Per-request resolution is simpler and always correct |
| Caching git user at startup | Defeats purpose if user changes config mid-session |
| Bundling both features into single flag | Different use cases, must be composable |
| Web UI for branch switching or author input | CLI/config only per project scope |
| Multi-branch simultaneous serving | Fundamentally different architecture |
| Parsing .gitconfig directly | `git config --get` handles layering correctly |
