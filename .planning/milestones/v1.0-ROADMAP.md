# Roadmap: Gollum Configuration Enhancements

## Overview

Two independent configuration options — `track_current_branch` and `local_git_user` — are added to Gollum's `wiki_options` hash, each exposed as both a config file key and a CLI flag. Branch tracking is implemented first (single integration point, visually verifiable), then local git user (two integration points in author paths), then a final validation phase ensures both options behave identically via config file and CLI, and that mutual exclusion runs after config file loading.

## Phase Details

### Phase 1: Branch Tracking
**Goal**: Wiki serves pages from whatever branch is currently checked out, updating dynamically without restart
**Depends on**: Nothing (first phase)
**Requirements**: BRANCH-01, BRANCH-02, BRANCH-03, BRANCH-04
**Success Criteria** (what must be TRUE):
  1. User sets `wiki_options[:track_current_branch] = true` in config.rb (or passes `--track-current-branch`), starts Gollum, switches branches with `git checkout`, and refreshing the wiki shows content from the new branch
  2. User sets both `--ref main` and `--track-current-branch` and Gollum exits immediately with a clear error message explaining the conflict
  3. User detaches HEAD (`git checkout <sha>`) while `track_current_branch` is active, and Gollum either serves the detached SHA or shows a clear error rather than crashing
  4. Existing `--ref` behavior is unchanged when `track_current_branch` is not set
**Plans:** 2/2 plans complete

### Phase 2: Local Git User
**Goal**: Web edits are attributed to the developer's local git identity (user.name / user.email) from git config
**Depends on**: Phase 1
**Requirements**: USER-01, USER-02, USER-03
**Success Criteria** (what must be TRUE):
  1. User sets `wiki_options[:local_git_user] = true` in config.rb (or passes `--local-git-user`), makes a web edit, and `git log` shows the commit authored by their local git config user.name/email
  2. User changes their git config user.name between two web edits (without restarting Gollum), and both commits reflect the correct name at the time of each edit
  3. User has no git user.name or user.email configured, and Gollum falls back gracefully (warning, not crash) rather than producing a broken commit

### Phase 3: Configuration Validation
**Goal**: Both options work identically whether set via config file or CLI, and mutual exclusion catches conflicts from any source
**Depends on**: Phase 2
**Requirements**: CONF-01, CONF-02
**Success Criteria** (what must be TRUE):
  1. User sets `track_current_branch` via config file and gets identical behavior to passing `--track-current-branch` on CLI (same for `local_git_user`)
  2. User passes `--track-current-branch` on CLI while config file sets `wiki_options[:ref] = 'main'`, and Gollum exits with the mutual exclusion error (validation runs after config file loading)
  3. User sets both flags via config file (no CLI flags at all) and both features work correctly together
**Plans:** 2 plans
